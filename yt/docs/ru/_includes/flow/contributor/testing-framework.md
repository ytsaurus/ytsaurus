# Фреймворк для тестирования пайплайнов {{product-name}} Flow

Страница описывает настройку рабочей машины и низкоуровневые параметры для контрибьюторов, дорабатывающих сам Flow и запускающих его интеграционный тестовый набор. Авторам пайплайнов, которые тестируют свой код, см. [Тестирование (C++)](../../../flow/cpp/testing.md), [Тестирование (Java)](../../../flow/java/testing.md) и [Тестирование (Python)](../../../flow/python/testing.md) — там собраны параметры запуска и приёмы, нужные для пайплайн-тестов.

## Подготовка окружения {#setup}

Тестовый набор Flow ловит падения процессов Flow и достаёт из coredump'ов бектрейсы. Чтобы это работало, на машине должен быть включён сбор coredump'ов:

```bash
# Временно (лучше прописать в ~/.bashrc).
ulimit -c unlimited
```

И добавление PID процесса в имя файла coredump:

```bash
# Временно — действует до перезагрузки:
sudo sysctl -w kernel.core_uses_pid=1

# Постоянно:
echo "kernel.core_uses_pid=1" | sudo tee -a /etc/sysctl.conf
sudo sysctl -p
```

Без `core_uses_pid=1` возможны странности вроде многократного извлечения бектрейсов из одного coredump.

Если для отладки нужен UI локального {{product-name}}, понадобятся сетевые доступы — см. [соответствующий раздел](../../../flow/cpp/testing.md#debug-ui-yt) в документации для авторов пайплайнов.

## Параметры тестов Flow {#test-param}

Помимо общих параметров (см. [Тестирование (C++)](../../../flow/cpp/testing.md#debug-test-framework)), при работе над самим Flow полезны параметры, влияющие на внутреннее поведение фреймворка:

#|
|| **Параметр** | **По умолчанию** | **Значения** | **Действие** ||
|| `DUMP_BACKTRACES_IF_HANGS` | `0` | `0`, `1` | Записывать на диск бектрейсы потоков и файберов запущенных процессов Flow при непойманном `WaitFailed` во время остановки. ||
|| `FLOW_BINARY_PATH` | | абсолютный путь | Использовать фиксированный бинарь пайплайна — например, собранный с thread-санитайзером. ||
|| `ERROR_BACKTRACE_ENRICHER_LEVEL` | `enabled_for_not_native_errors` | [см. enum]({{source-root}}/yt/yt/flow/cpp/misc/error_backtrace_enricher.h) | Управляет добавлением бектрейсов к ошибкам в логах. ||
|#

Примеры:

```bash
ya make -A --test-param ERROR_BACKTRACE_ENRICHER_LEVEL=enabled_for_all
ya make -A --test-param DUMP_BACKTRACES_IF_HANGS=1
```
