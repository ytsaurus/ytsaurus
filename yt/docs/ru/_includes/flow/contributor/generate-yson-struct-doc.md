# Генератор документации по YSON-структурам {{product-name}} Flow

Часть пользовательской документации {{product-name}} Flow генерируется напрямую из исходного кода — это позволяет держать описание YSON-структур синхронным с C++-определениями без ручной поддержки.

Скрипт собирает markdown-описание YSON-структур, используемых для конфигурации {{product-name}} Flow. Помимо сводного [Справочника конфигураций](../../../flow/generated_docs/all_yson_structs.md) генератор создаёт по отдельному файлу на каждую корневую и вложенную структуру (а также на встречающиеся enum-ы и типы вроде `TDuration`) — все они лежат рядом, в директории `generated_docs/`.

Источники данных:
- Метаданные из C++-кода (имена и типы параметров, дефолтные значения и т. д.), извлечённые непосредственно из YSON-структур.
- Markdown-описания структур и их полей, написанные руками разработчиками и техписателями в `description_ru.yaml`.

## Как внести правку в сгенерированную документацию {#edit}

Поправить `description_ru.yaml`, запустить генератор, закоммитить изменения.

## Запуск генератора {#run}

После изменений в коде YSON-структур или в `description_ru.yaml`:

```bash
cd $(arc root)/yt/yt/flow/tools/generate_yson_struct_doc
./generate.sh
```

Скрипт сам сделает нужные `arc add` на сгенерированную документацию. После этого можно коммитить.

Английские описания находятся в файле `description_en.yaml`, английская документация генерируется командой `./generate.sh en` и попадает в `yt/docs/en/flow/generated_docs`. При изменении описаний старайтесь поддерживать в актуальном состоянии оба файла: тест `test_check_all_description_fields_exist` прогоняется отдельно для `description_ru.yaml` и `description_en.yaml` и для каждого из них проверяет, что перечисленные в нём структуры, поля и значения enum-ов существуют в коде, — он не сверяет файлы между собой и не заметит описание, изменённое или пропущенное только в одном из языков. Английская документация пока не включена в `toc.yaml`.

## Добавление новых корневых YSON-структур {#new-roots}

Генератор берёт несколько корневых YSON-структур (`TPipelineSpec`, `TFlowNodeConfig`, параметры компьютейшенов, сорсов, синков, …), рекурсивно обходит их по вложенным полям и извлекает все найденные схемы.

Если новая структура недоступна из текущих корневых, её нужно явно добавить в `./bindings/module.cpp` — иначе генератор её не увидит.
