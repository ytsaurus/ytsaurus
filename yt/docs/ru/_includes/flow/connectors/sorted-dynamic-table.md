# Sorted Dynamic Table Sink в {{product-name}} Flow

Коннектор для записи данных в [сортированные динамические таблицы {{product-name}}](../../../user-guide/dynamic-tables/sorted-dynamic-tables.md).

{% note info "Внимание" %}

В большинстве случаев для работы с сортированными динамическими таблицами следует использовать [StateManager](../../../flow/concepts/stateful.md), а не этот коннектор.

Основные отличия:

- **StateManager** поддерживает read-modify-write операции, удаление строк по нетривиальным правилам и гарантирует единую точку модификации каждой строки. `group_by_schema` computation должна совпадать (или быть префиксом) с ключевыми колонками стейта.
- **Этот коннектор** &mdash; низкоуровневый примитив, поддерживающий только write-only запись (вставку или удаление строк). Ограничений на `group_by_schema` нет, что позволяет писать в таблицу из нескольких computation с разными ключами, но при этом порядок модификаций одной строки не контролируется.

{% endnote %}

Код коннектора находится [здесь]({{source-root}}/yt/yt/flow/library/cpp/connectors/sorted_dynamic_table).

## Запись в сортированную динамическую таблицу

Синк принимает сообщения и записывает их строки в сортированную динамическую таблицу. Схема сообщений должна быть совместима со схемой целевой таблицы.

Запись выполняется синхронно в рамках основной транзакции эпохи (`NYT::NFlow::NSortedDynamicTable::TSyncSink`). Это означает, что запись атомарна вместе с остальными изменениями эпохи, но таблица должна находиться на основном кластере процессинга &mdash; на том же, что и пайплайн.

Количество каналов получателя (receiver channels) автоматически подстраивается под количество таблетов целевой таблицы &mdash; контроллер периодически опрашивает таблицу и обновляет это значение.

### Параметры синка

Класс синка: `NYT::NFlow::NSortedDynamicTable::TSyncSink`.
