# Коннекторы {{product-name}} Flow

[Коннектор](../../../flow/concepts/glossary.md#connector) — компонент, связывающий пайплайн с объектами {{product-name}} (очередью, таблицей и т. д.). Каждый коннектор предоставляет [сорс](../../../flow/concepts/glossary.md#source) для чтения сообщений и/или [синк](../../../flow/concepts/glossary.md#sink) для записи.

{% note info %}

Используемые пайплайном коннектеры непосредственно влияют на предоставляемые им гарантии обработки сообщений. Перед выбором коннекторов важно ознакомиться с разделом [Гарантии обработки](../../../flow/concepts/guarantees.md).

{% endnote %}

{% if audience == "internal" %}Интеграции с внешними (не-{{product-name}}) системами описаны в разделе [Расширения](../../../yandex-specific/flow/extensions/about.md).{% endif %}

## Смена источника {#source-change}

Сорс идентифицирует свои [партиции](../../../flow/concepts/glossary.md#partition) по параметрам, описывающим физический источник; конкретный набор таких параметров указан в документации соответствующего коннектора. Эти параметры входят в ключ партиции, поэтому при их изменении сорс описывает уже другой набор партиций.

При смене источника партиции старого источника исчезают из набора и **завершаются** (`Completing` ⇒ `Completed`): связанный с ними [стейт](../../../flow/concepts/glossary.md#state), включая запомненные оффсеты, удаляется, а не сохраняется в расчёте на возврат той же партиции (как при интеррапте во время репартиционирования). Партиции нового источника создаются с нуля.

{% note warning %}

Параметры, идентифицирующие источник, входят в статический [`Spec`](../../../flow/concepts/spec.md), поэтому менять их можно только на остановленном пайплайне. Новый источник читается независимо от того, что уже было обработано старым: если он содержит те же данные, они будут записаны в выходные потоки повторно.

{% endnote %}
